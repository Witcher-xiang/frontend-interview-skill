# 评分标准与反馈模板

## 评分维度

| 维度 | 权重（初级） | 权重（中级） | 权重（高级） |
|------|-------------|-------------|-------------|
| 前端基础 | 30% | 20% | 15% |
| 框架深度 | 20% | 25% | 25% |
| 工程化与架构 | 15% | 25% | 30% |
| 项目经验 | 15% | 15% | 20% |
| Vibe Coding / AI 工具 | 10% | 5% | 5% |
| 思维逻辑与表达 | 10% | 10% | 5% |

> 高级权重偏向工程化与架构、项目经验（生产级）；初级权重偏向前端基础和学习潜力。

## 各风格反馈措辞示例

同一份面试结果，六种风格会给出不同的反馈措辞。以下是中级前端同一场面试后的反馈对比：

### 严厉拷打型

```
综合评分 5.5/10。

前端基础 5.0：闭包和原型链能说清楚，但问到 Event Loop 的微任务/宏任务执行顺序就开始模糊了，
说明理解还是停在表面。

框架深度 5.5：React Hooks 会用，但问 Fiber 架构为什么用链表就答不上来了。只会用不懂原理，
在面试中很吃亏。

工程化与架构 5.0：Vite 在用，但"为什么比 Webpack 快"说不清楚，只会说"快"。

项目经验 6.0：项目在跑，但没有性能数据。首屏加载时间、Lighthouse 分数、Core Web Vitals 指标
一个都说不出来。

Vibe Coding / AI 5.0：Copilot 在用，但说不出具体哪些场景用 AI 效率高、哪些场景反而浪费时间。

你现在出去面大概率挂。主要问题是框架只停在使用层不理解原理 + 项目缺乏量化数据。建议：
1. 把 Event Loop 和 Fiber 架构搞透，能画出完整流程图
2. 给你的项目跑一次 Lighthouse，整理出性能基线数据
3. 上面两条没做到之前别投了。
```

### 温和鼓励型

```
综合评分 5.5/10，整体表现中规中矩，有一些亮点也有一些需要提升的地方。

前端基础 5.0：闭包和原型链的基础还是比较扎实的，这些核心概念讲得不错。
如果能把 Event Loop 的细节再深入一些就更好了。

框架深度 5.5：React Hooks 用得挺熟练的，能看出来实际项目中有不少经验。
建议接下来可以花点时间了解 Fiber 架构的设计思路，会帮助你理解很多性能优化的细节。

工程化与架构 5.0：有在用 Vite，这是好的开始。建议进一步了解它的原理。

项目经验 6.0：你有实际项目经历，这在中级前端中是不错的。
如果能整理出一些性能指标就更有说服力了。

Vibe Coding / AI 5.0：看得出来你有在用 AI 工具辅助开发，继续保持。

建议下一步：1. 深入学习 Event Loop 和框架原理；2. 整理项目中的性能数据。
总体来说基础不错，继续加油！
```

### 专业高效型

```
综合评分 5.5/10。

【结论】前端基础基本合格，框架深度待加强，工程化偏弱。

前端基础 5.0/10 — 闭包 6分，Event Loop 4分（微任务顺序不清）
框架深度 5.5/10 — Hooks 使用 7分，原理 4分（Fiber 不了解）
工程化与架构 5.0/10 — 构建工具在用，原理不清
项目经验 6.0/10 — 有项目经历，缺乏性能数据支撑
Vibe Coding / AI 5.0/10 — AI 工具使用较浅

【核心短板】框架原理理解不足；项目缺少量化数据；工程化深度不够。

【下一步建议】
  1. 系统学习 Event Loop（重点：微任务/宏任务执行顺序）
  2. 阅读 React Fiber 架构文档或源码解析文章
  3. 用 Lighthouse 和 Web Vitals 量化项目性能指标
```

### 深挖学术型

```
综合评分 5.5/10。

你的思维路径比较清晰，但对一些核心机制的理解还停留在"使用层面"，
缺乏"设计层面"的视角。这是中级前端常见的瓶颈，需要从"会用"升级到"理解为什么这样设计"。

前端基础 5.0：你描述了闭包的行为，但没有进一步分析 V8 引擎如何处理闭包中的变量引用、
为什么闭包能访问外部作用域的变量。这些问题帮助你理解 JS 引擎的设计权衡。

框架深度 5.5：React Hooks 会用但不知道为什么要这样设计——
为什么用链表？为什么不能条件调用？这些设计约束背后有具体的工程原因。

工程化 5.0：知道 Vite 快，但不知道"快在哪里"。建议从浏览器原生 ESM 的角度理解。

建议下一步：阅读 Dan Abramov 关于 React Hooks 设计的文章，
以及 Evan You 关于 Vite 设计哲学的演讲，从设计者视角理解框架选择。
```

### 工程实践型

```
综合评分 5.5/10。

你的优势是有实际项目经历，这是很多同级别候选人缺乏的。但项目的技术深度不够。

前端基础 5.0：能讲清楚基本原理，但结合生产场景就卡住了。
比如你说用到了懒加载，但我问你首屏加载优化前后的数据对比，你说"应该快了"——
这就是没有量化的表现。

框架深度 5.5：React 在用，这很好。但我需要数据：组件渲染次数优化了多少？
用了哪些手段？memo 是随便加的还是有性能分析支撑？

项目经验 6.0：项目在跑。但我问首屏 LCP 是多少、Lighthouse Performance 得分是多少、
打包体积是多少——一个都说不出来。在生产环境会很被动。

建议下一步：
1. 给你的项目跑一次 Lighthouse，记录各项指标基线
2. 用 React DevTools Profiler 找到渲染瓶颈并优化
3. 了解一下前端监控体系（Sentry/自建方案）
```

### 综合平衡型

```
综合评分 5.5/10。

今天的面试表现中规中矩，有一些做得不错的地方，也有明显的提升空间。

前端基础 5.0：JS 基础是达标的，闭包、原型链都能讲清楚。
不过 Event Loop 是薄弱点，建议重点看一下微任务/宏任务的执行顺序。

框架深度 5.5：React Hooks 使用挺熟练的，这是加分项。
不过框架原理还需要更深入，比如 Fiber 架构和 Reconciliation 的机制。

工程化 5.0：构建工具在用但理解不深，建议了解 Vite 的原理。

项目经验 6.0：有实际项目经历，不过技术细节还需要更深入。
能说出具体的性能数据和技术选型的权衡会更好。

Vibe Coding / AI 5.0：AI 工具在用，建议深入了解一些高级用法。

下一步建议：
1. 深入 Event Loop 原理，重点理解微任务/宏任务执行顺序
2. 学习 React Fiber 架构，理解 Virtual DOM 和 Reconciliation
3. 用 Lighthouse 量化项目性能，建立数据思维

今天的面试是一次很好的练习，继续准备，下次会更好。
```

| 遇到困难 | "这个问题我在生产环境遇到过，大概是这么个情况……你想想你会怎么排查？" |
| 回答较好 | "好，那你们有没有遇到过大量组件重渲染的性能问题？怎么处理的？" |
| 压力测试 | "你说你做过性能优化，那优化前后的 Lighthouse 分数分别是多少？" |

### 综合平衡型

| 场景 | 话术 |
|------|------|
| 回答模糊 | "你说的方向是对的。能再具体一点吗？比如你在这个点具体做了什么？" |
| 遇到困难 | "没问题，这个点有难度。我换一种问法……你觉得这个和 XX 有什么关系？" |
| 回答较好 | "很好，这个点理解到位了。顺着这个我想多问一层……" |
| 压力测试 | "好，那我们延伸一下：如果场景变成……你会怎么考虑？"（讨论式，不说"不够"）|

### 通用（所有风格通用）

| 场景 | 话术 |
|------|------|
| 时间紧张时 | "由于时间关系，这个问题先到这里。" |
| 连续两次答不上来 | "提示一下：这个问题涉及 XX 原理/机制。"（给出一个方向的提示，不直接给答案）|
