# 前端开发面试模拟器

使用自然语言与 AI 面试官互动，模拟真实前端开发岗位面试。支持按求职者身份、技术栈和面试官风格定制，全程记录状态并输出结构化评估报告。

---

## 核心特性

**多身份模拟** — 针对不同阶段候选人设计差异化问题：初级前端（0-1 年）、中级前端（1-3 年）、高级前端（3-5 年）。难度和追问深度随身份自动调整。

**技术栈适配** — 支持 React 和 Vue 两大主流框架，面试内容根据候选人选择的技术栈动态调整。两者都熟悉的候选人会被交叉提问。

**六种面试官风格** — 从"严厉拷打"到"温和鼓励"，从"工程实践"到"深挖学术"，覆盖不同偏好。同一场面试，不同体验。

**实时状态跟踪** — 面试全程维护结构化状态（已考察点、候选人弱点、待跟进项），避免重复提问，确保评估连贯。

**简历与 JD 匹配** — 可选提供简历（支持 PDF）和目标岗位 JD，面试内容自动适配岗位要求，最终报告包含匹配度分析和针对性备战建议。

**高频技术题库** — 覆盖 HTML/CSS、JavaScript、TypeScript、React/Vue、前端工程化、浏览器与网络、系统设计、Vibe Coding 与 AI 工具等核心方向。

**完整评估反馈** — 面试结束后输出综合评分、各维度评分及风格化改进建议，反馈措辞随面试官风格变化。

## 快速开始

### 环境要求

- [Claude Code](https://claude.com/code)（桌面端）
- Node.js 18+

### 安装方式

将 `SKILL.md`、`README.md` 和 `references/` 目录复制到 AI 平台的 skills 目录即可使用。

### 基本使用

在支持 Skills 的 AI 平台中直接表达面试意图即可触发：

```
开始一场前端模拟面试
我是中级前端，主要用 React，想练习面试，温和鼓励型
```

也可以在开始时直接提供简历和目标岗位：

```
开始面试，这是我的简历 ~/Documents/resume.pdf，目标岗位 JD 如下：
[粘贴目标岗位描述]
```

系统会依次确认：求职者身份、简历与 JD（可选）、技术栈偏好、面试官风格、面试时长。

### 语音输入（推荐）

## 进阶用法

### 控制命令

面试过程中可使用以下命令：

- **继续** — 进入下一环节
- **跳过** — 跳过当前问题
- **结束** — 提前结束并生成报告
- **换个风格** — 实时切换面试官风格
- **反馈** — 查看当前评分

### 简历与 JD 支持

提供简历和目标岗位 JD 可获得高度定制化的面试体验：

- **简历**：直接粘贴简历文本，或提供文件路径（支持 PDF 格式）
- **目标岗位 JD**：粘贴目标职位的岗位描述文本

提供后，面试问题将围绕 JD 要求调整，项目追问聚焦简历亮点验证，最终报告包含岗位匹配度分析和针对性备战建议。

> 两项均为可选，提供任意一项即可获得更有针对性的面试体验。

## 技术架构

本项目基于 **OpenClaw Skill 框架**构建，采用三层加载架构：

| 层级 | 文件 | 作用 |
|------|------|------|
| **元数据** | `SKILL.md` frontmatter | 触发条件与能力概述，始终加载 |
| **主体逻辑** | `SKILL.md` body | 面试流程、风格适配、状态跟踪规则 |
| **知识库** | `references/*.md` | 按需加载：技术题库、评分标准、风格指南 |

面试流程采用**有限状态机**思路：破冰 → 简历/JD 分析（可选）→ 项目介绍 → 前端基础 → 框架深度 → 工程化与系统设计 → Vibe Coding / AI → 总结反馈，各阶段时长和深度由身份和剩余时间共同决定。

## 当前局限

- **面试时长依赖估算**：实际面试中时间分配是动态的，本项目按参考时长设计，但不做精确计时
- **不支持实时语音**：当前为文字交互，暂不涉及口语表达评估（表达能力通过文字间接评估）
- **题库覆盖有边界**：主要覆盖前端开发高频方向（HTML/CSS/JS/TS/React/Vue/工程化/Vibe Coding），后端开发方向暂不涉及
- **简历格式支持**：支持文本和 PDF 格式简历，暂不支持图片格式简历（如扫描件、截图）
